Prototype versioning

Multiple versions in a single app

  • Each version needs it's own folder in 'views'
  • Current is the latest version, duplicate this when parking a version
  • Each version requires it's own routes file, which stores the version number and version specific routing
  • You need to include each version routes file in the master routes.js
  • Hyperlinks can use relative pathing (/, ./, ../) to maintain the current version
  • You can pass the version back to the template via a res.get, for js functions, routing, dynamic links etc
